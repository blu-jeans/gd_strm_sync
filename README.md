这是一个基于 FastAPI + Tailwind UI (Sleek Dark Mode) 打造的 **STRM 媒体同步与控制面板**。它能够将谷歌云盘等远程挂载目录中的视频元数据，实时转换为 `.strm` 格式文件并挂载分发给 Emby。除了视频文件外，还支持将网盘内字幕（ass/srt）、海报、主题音乐及 nfo 信息等元数据原样增量同步拷贝到本地。

本项目已被完全容器化，支持 **AMD64 / ARM64** 多架构构建，确保在 PC、群晖/威联通 NAS、软路由、树莓派等千差万别的运行环境中均可高效运行。

## 🚀 部署指南

在服务器上运行该系统，只需准备一个 `docker-compose.yml` 配置文件即可直接运行。

### 1. 准备工作目录
在宿主机上创建一个空文件夹（例如 `/root/strm-sync`），并在该文件夹内新建 `docker-compose.yml`。

### 2. 编写 `docker-compose.yml`
```yaml
version: '3.8'

services:
  strm-sync-panel:
    image: fckipk/strm-sync-panel:latest
    container_name: strm-sync-panel
    restart: always
    ports:
      - "9000:8000"
    environment:
      # 宿主机上 emby 用户的 UID 和 GID，解决容器内外权限冲突，请根据 id emby 的实际结果进行替换
      - EMBY_UID=998
      - EMBY_GID=997
      # 2. 动态指定容器内可安全加载的挂载卷范围 (需与 volumes 保持一致)
      - GD_ROOT=/mnt          
      - STRM_ROOT=/mnt/strm
    volumes:
      # 挂载后，宿主机目录下会自动产生 cache/ 数据库和 logs/ 滚动日志
      - ./cache:/app/cache
      - ./logs:/app/logs
      # 挂载宿主机 rclone 配置文件目录 (只读)，确保容器内能识别远程网盘
      - /root/.config/rclone:/root/.config/rclone:ro
      # 挂载宿主机 rclone 的 Service Account 证书目录 (只读)，防止 accounts 文件找不到，没使用SA的可以忽略此配置
      - /root/accounts:/root/accounts:ro
      # 挂载宿主机的媒体根目录与 strm 目的输出目录 (视实际路径修改)
      - /mnt:/mnt
```

### 3. 启动服务
在当前工作目录下执行以下命令：
```bash
docker-compose up -d
```
启动后，访问 `http://<你的服务器IP>:9000` 即可登录控制面板进行可视化操作。


## ⚙️ 核心环境变量与配置项说明

系统通过环境变量来进行自适应运行调整，关键配置项如下：

| 环境变量 | 默认值 | 说明 |
| :--- | :--- | :--- |
| `EMBY_UID` | `998` | 宿主机上 Emby 用户的 UID。为解决容器内外权限冲突，**请在宿主机执行 `id emby` 获取实际结果并进行替换**。新生成的 `.strm` 文件与目录会被自动赋予该 UID 所有权。 |
| `EMBY_GID` | `997` | 宿主机上 Emby 用户的 GID。**请在宿主机执行 `id emby` 获取实际结果并进行替换**。 |
| `GD_ROOT` | `/mnt` | 远程网盘在本地映射的根路径白名单。面板只允许访问和同步在此根路径之下的目录，以防路径越权。 |
| `STRM_ROOT` | `/mnt/strm` | 生成 `.strm` 媒体文件的本地挂载输出目标目录。 |

## 📁 数据持久化与日志安全

* **数据库持久化**：SQLite 数据库保存在 `/app/cache/strm_sync.db` 中。在容器部署时，已将其映射到宿主机的 `./cache`。即使升级镜像或重构容器，所有同步任务、设置与网盘配置依然**保持原样不会丢失**。
* **按天滚动日志与防撑爆机制**：系统日志存储于 `/app/logs/` 下。为了防止服务器磁盘空间被无限增长的日志写满，后台日志记录器集成了**滚动备份与过期自动清理机制**（可在系统中指定最大保留体积/天数，超期将自动删除旧日志），保障生产环境的安全稳定。

## ✨ 新增功能特性：元数据增量同步 (Metadata Sync)
- **原样直接下载**：除了视频转为 `.strm` 占位外，网盘中与之配对的元数据（如外挂字幕 `.ass/.srt`、海报 `.jpg/.png`、描述文件 `.nfo` 等）将增量拷贝下载至本地，且完全保持云端的层级目录与原名。
- **自主可控扩展**：您可在 **系统设置** 里自定义元数据类型扩展名（默认涵盖 `nfo,jpg,jpeg,png,svg,ass,srt,sup,mp3,flac,wav,aac`）；同时在 **同步源配置** 里，可对个别数据源单独关闭/开启元数据同步（默认开启）。
- **智能差分与孤儿清理**：元数据同步为全自动增量差分（已存在且无变化的文件自动跳过），当网盘中元数据被删除时，本地对应的元数据也会自动抹除。若目录下既无视频也无有效元数据，系统将自动回收空文件夹（孤儿目录），防止冗余数据堆积。

为了让界面更加专业，本项目针对当前主力支持的谷歌云盘（Google Drive）进行了全套深度设计：
- **谷歌云盘独占显示**：系统侧边栏分类及弹窗的网盘类型选择，目前仅优先显示 Google Drive，屏蔽了其他未适配网盘（如 115、OneDrive、阿里云盘等）。其他网盘在代码中通过注释保留，以便未来适配后直接恢复。
- **官方彩色 SVG 徽标**：Google Drive 菜单及同步源卡片中的徽标，均已集成与官方完全一致的高保真彩色三角形 SVG 图标，极大提升控制面板的质感，且在亮色/暗黑模式下均能自适应完美显示。
- **拖拽排序与置顶完备**：在“全部源”以及“Google Drive”过滤模式下，均完美支持点击卡片进行物理拖拽排序或一键置顶，并在操作完成后自动同步提交保存至后端 SQLite3 数据库中的 `sort_order`，实现用户自定义卡片位置持久化。
- **Mock 测试源智能隔离**：本地 SQLite 数据库在初始化时，如果检测到运行于 Windows 环境（本地开发测试），会自动载入 3 个测试同步源供拖拽及布局测试；若检测到运行于 Linux Docker 容器环境（生产服务器部署），则自动保持库表干净空置，不会写入任何脏测试数据。
- **一键打包推送辅助**：项目根目录下新增了 Windows PowerShell 脚本 `build_and_push.ps1` 和 Linux Shell 脚本 `build_and_push.sh`。测试通过后直接双击或执行脚本，即可在终端以交互式菜单完成**单架构极速本地打包**或**多架构 Docker Buildx 交叉编译（AMD64 & ARM64）**并一键推送到 Docker Hub 仓库。
